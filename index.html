Guia operacional para subir APIs Flask/Django/FastAPI usando Gunicorn no Host e Nginx no Docker.

1. Clone & Preparação

Regra de Ouro: NUNCA use a pasta venv do Windows. Crie uma nova no Linux.

Acesse a pasta de infraestrutura e prepare o ambiente:

# 1. Acesse a pasta central e clone
cd ~/infra-proxy
git clone https://github.com/SEU_USUARIO/NOVO-PROJETO.git
cd NOVO-PROJETO

# 2. Cria e ativa o ambiente virtual nativo
python3 -m venv venv
source venv/bin/activate

# 3. Instala dependências + Gunicorn
pip install -r requirements.txt
pip install gunicorn

# 4. Crie o arquivo de senhas
nano .env

2. Serviço Systemd

Crie o arquivo /etc/systemd/system/meu-app.service. Defina uma porta única.

[Unit]
Description=Gunicorn instance for MEU_APP
After=network.target

[Service]
User=root
WorkingDirectory=/root/infra-proxy/MEU_APP
Environment="PATH=/root/infra-proxy/MEU_APP/venv/bin"

# ⚠️ ATENÇÃO NA PORTA (Ex: 5001, 5002...)
ExecStart=/root/infra-proxy/MEU_APP/venv/bin/gunicorn --workers 3 --bind 0.0.0.0:5001 app:app

[Install]
WantedBy=multi-user.target
sudo systemctl start meu-app sudo systemctl enable meu-app

3. Configuração Nginx (Docker)

Edite o conf/default.conf. Aponte para o Gateway do Docker (172.17.0.1) na porta definida acima.

server {
    listen 80;
    server_name app.99dev.pro;

    location /.well-known/acme-challenge/ { root /var/www/certbot; }

    location / {
        # O IP 172.17.0.1 permite o Docker falar com o Host
        proxy_pass http://172.17.0.1:5001; 
        
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
docker exec porteiro-nginx nginx -s reload

Gestão Operacional

Controle de Portas (Mental Model)

Cada projeto Gunicorn precisa de uma porta exclusiva no servidor Host. O Nginx atua como telefonista direcionando para o ramal correto.

Projeto Porta (Systemd) Domínio (Nginx)
api-atas 5000 api.99dev.pro
novo-projeto 5001 novo.99dev.pro
dashboard 5002 dash.99dev.pro

Gerar Cadeado SSL

Após configurar o Nginx na porta 80, rode o Certbot via Docker:

docker run -it --rm --name certbot \
-v "/root/infra-proxy/certbot/conf:/etc/letsencrypt" \
-v "/root/infra-proxy/certbot/www:/var/www/certbot" \
certbot/certbot certonly --webroot -w /var/www/certbot \
-d SEU.DOMINIO.COM

*Depois mude o Nginx para listen 443 ssl.
